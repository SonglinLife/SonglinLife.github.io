# MongoDB 数据转 compute

MongoDB metas 数据格式：

需要数据：

表一：记录包基本数据 （ods_npm_package_basic）

  • name

  • status

  • lastUpdatedAt

  • nextUpdateAt

  • created

  • modified

表二：版本记录 （ods_npm_package_version)

  • name
  • version
  • time

npm 元数据：

查询 npm_version：22425828 条数据

version 里有 1823843 条不同名称版本的数据

查询 npm_basic：1833159 条数据，也就是 1833159 个仓库，为什么有仓库丢失了？

这些仓库没有 releases 字段，这个是空的

我终于知道为什么会有 release 丢失了，unwind，如果没有

表三： records_basic 表

现在大致有这些好用的数据

  • name
  • version
  • detail.name
  • detail.version
  • detail.description
  • detail.keywords (array)
  • detail.homepage
  • detail.main
  • detail.author.name
  • detail.license
  • detail.dependencies (dict)
  • detail._id
  • detail._npmVersion
  • detail._nodeVersion
  • detail._npmUser.name
  • detail._npmUser.email
  • detail.dist.tarball
  • detail.dist.fileCount
  • detail.dist.unpackedsize
  • detail.maintainers.name
  • detail.maintainers.email

想做的表应该含有的数据：

  • name
  • version
  • description
  • keywords （这两个都要分开搞）
  • homepage
  • main
  • license
  • detail.dependencies (dict) (最重要的部分)
  • nodeVersion
  • npmVersion
  • fileCount
  • unpackedSize

表 ods_npm_records：

把不需要 unwind 的数据先提取出来。

22453592 条数据

# 表 dwd_npm_dependency

这里用 "$objectToArray 先来将 dict 也就是 document 转换为 array，再用 $unwind 来让 array 展开，最后展开的会是 k、v 的键值对，用二级索引来处理。
